Supported Communication Protocols

This chapter provides information about supported communication protocols.

High Speed LAN Instrument Protocol (HiSLIP)

  • The protocol is available on TCP port 4880. To use this protocol the user needs a VISA library.
     
    Command termination: The input SCPI command can have a '\r\n', '\r', '\n' or no termination. For query commands the result data will always have a '\n' termination.

RAW TCP

  • The protocol is available on TCP port 5025. Putty can be used to send commands commands over the raw interface.
     
    Command termination: The input SCPI command termination '\r\n', '\r' and '\n' we be used by the query result. When the command has no termination the query result will always use '\n'.

RAW UDP

  • The protocol is available on UDP port 5025. This is a special implementation, allowing to get measurements over UDP. The set of SCPI commands are limited.
     
    Command termination: The input SCPI command don't need termination. For query commands the result data will always have a '\n' termination.

SCPI Commands Documentation

This chapter provides the device SCPI (Standard Commands for Programmable Instruments) commands.

Basic Commands

Commands available in HiSLIP and RAW TCP.

  • *IDN?
    Queries the instrument's identity (manufacturer, model, serial number, and firmware version).
  • *RST
    This reset (RST) command resets most functions to factory-defined conditions.
  • *CLS
    Clears the status data structures.
  • *TRG
    Triggers the instrument.

Measurement Commands

Commands available in HiSLIP and RAW TCP.

  • MEASure?
    Measures AC or DC current.
  • READ?
    Measures AC or DC current.
  • R?
    Measures AC or DC current.
  • FETCh?
    Read stored measurement data. This command will not start a new measurement.
  • DATA[:DATA]? <data_index>,<data_count>
    Read segments of stored measurements.

    <data_index> - data index, can be from 0 to 31,999.
    <data_count> - data count, can be from 1 to 32,000.
  • INITiate[:IMMediate]
    Start measurements. You can store up to 32,000 measurements in the reading memory.
    To retrieve the measurements from the reading memory, use FETCh? or DATA[:DATA]?.

Configuration Commands

Commands available in HiSLIP and RAW TCP.

  • CONFiguration:RESOLution <resolution_value>
    CONFiguration:RESOLution?
    Configure the ADC resolution.

    <resolution_value> : { 8 | 10 | 12 | 14 | 16 } Default : 16
  • CONFiguration:SAMPling:TIME[:CYCLe] <sampling_cycle>
    CONFiguration:SAMPling:TIME[:CYCLe]?
    Configure the ADC sampling time using ADC cycles.

    <sampling_cycle> : { 1.5 | 2.5 | 8.5 | 16.5 | 32.5 | 64.5 | 387.5 | 810.5 } Default : 1.5
  • CONFiguration:OVERSAMPling[:ENAble] {OFF|ON}
    CONFiguration:OVERSAMPling[:ENAble]?
    Enable/disable the ADC oversampling function.
  • CONFiguration:GAIN <gain>
    CONFiguration:GAIN?
    Select the analog frontend gain value.

    <gain> : { 1 | 10 | 100 | } Default : 1
  • SAMPle:COUNt {<count>|MIN|MAX|DEF}
    SAMPle:COUNt?
    Select the analog frontend gain value.

    <count> : From 1 to 32,000 samples. Default : 1000
    MIN : 1
    MAX : 32000
    DEF : 1000

Calculate Commands

Commands available in HiSLIP and RAW TCP.

  • CALCulate:AVERage?
    Get the average value.

    NOTE : The average will be calculated from measurements available in memory. This command will not start the measurement. The user need to call INITiate[:IMMediate] or *TRG to get the value for calculations.
  • CALCulate:OFFSet:ENAble {OFF|ON}
    CALCulate:OFFSet:ENAble?
    Enable/disable offset calculation. When enabled, the calculated offset value will be subtracted from the measurements.
  • CALCulate:OFFSet
    CALCulate:OFFSet?
    When enabled (CALCulate:OFFSet:ENAble) the command will start a measurement to calculate the offset value.

    WARNING : This step will overwrite the measurement memory.

ADC Commands

Commands available in HiSLIP and RAW TCP.

  • ADC:NULL:OFFSet:ENAble {OFF|ON}
    ADC:NULL:OFFSet:ENAble?
    Enable/disable the ADC offset calculation. When enabled, the calculated offset value will be subtracted from the measurements.
  • ADC:NULL:OFFSet
    ADC:NULL:OFFSet?
    When enabled (ADC:NULL:OFFSet:ENAble) the ADC offset measurement is done by connecting the ADC input to ground.

System Commands

Commands available in HiSLIP and RAW TCP.

  • SYSTem:COMMunicate:LAN:IPADdress "<address>"
    SYSTem:COMMunicate:LAN:IPADdress? [{CURRent|STATic}]
    Assings a static Internet Protocol (IP) v4 address for the device.

    "<address>" : "nnn.nnn.nnn.nnn" there nnn can be value from 000 to 255, for example "192.168.1.123".
  • SYSTem:COMMunicate:LAN:SMASk "<netmask>"
    SYSTem:COMMunicate:LAN:SMASk? [{CURRent|STATic}]
    Assigns a subnet mask for the device to use in determining whether a client IPv4 address is on the same local subnet.

    "<netmask>" : "nnn.nnn.nnn.nnn" there nnn can be value from 000 to 255, for example "255.255.255.0".
  • SYSTem:COMMunicate:LAN:GATEway "<address>"
    SYSTem:COMMunicate:LAN:GATEway? [{CURRent|STATic}]
    Assigns a default gateway for the device. The specified IPv4 Address sets the default gateway, which allows the instrument to communicate with systems that are not on the local subnet.

    "<address>" : "nnn.nnn.nnn.nnn" there nnn can be value from 000 to 255, for example "192.168.1.1".
  • SYSTem:COMMunicate:LAN:MAC?
    Returns the Media Access Control (MAC) address as an ASCII string of 12 hexadecimal characters (0-9 and A-F). The output format is "XX:XX:XX:XX:XX:XX".
  • SYSTem:COMMunicate:LAN:UPDate
    Write the new network configuration.

    NOTE : The device needs to be unsecured by using the SYSTem:SECure:STATe command.
  • SYSTem:SERVice:MDNS[:ENAble] {OFF|ON}
    SYSTem:SERVice:MDNS[:ENAble]?
    Enable/disable the multicast DNS (mDNS) service. The mDNS is used to autodetect device with VISA software.

    NOTE : The device needs to be unsecured by using the SYSTem:SECure:STATe command. The setup will be stored in the device's EEPROM and will be permanent.
  • SYSTem:SERVice:HISLIP[:ENAble] {OFF|ON}
    SYSTem:SERVice:HISLIP[:ENAble]?
    Enable/disable the High Speed LAN Instrument Protocol (HiSLIP) service. It is a TCP/IP-based protocol for remote instrument control of LAN-based test and measurement instruments.

    NOTE : The device needs to be unsecured by using the SYSTem:SECure:STATe command. The setup will be stored in the device's EEPROM and will be permanent.
  • SYSTem:SERVice:LED[:ENAble] {OFF|ON}
    SYSTem:SERVice:LED[:ENAble]?
    Enable/disable the device status LED. When enabled, the LED blinks every 2 seconds with a short green light and changes to a constant blue color when SCPI commands are being handled. When disabled, the status LED remains off. Default : enabled

    NOTE : Disable status doesn't influence SYSTem:SERVice:LED:PING command. This setup is not permanent.
  • SYSTem:SERVice:LED:PING
    This command will blink the status LED with a white light. It can be used to identify a device mounted in a system with other similar cards.
  • SYSTem:ERRor[:NEXT]?
    This command gets error code & description.
  • SYSTem:ERRor:COUNt?
    Returns the number of errors contained in the error-message buffer.
  • SYSTem:SERVice:RESET
    Perform a hardware reset of the device.

    WARNING : The connection to the device will be lost.

Format Commands

Format the result data.

  • FORMat[:DATA] {ASCii|REAL}
    FORMat[:DATA]?
    Set and query the format (binary or ASCii) by which data is read.

    ASCii - data is transferred in a human-readable ASCII format.
    REAL - data is transferred in block format. The block format consist of <block header> <block data> <termination>.

    Block header
    The header consists of a number of characters indicating the length of the data block. The typical format is #<n><nnnn...>, where <n> is the number of digits in the length field and <nnnn...> is the actual length of the data block in bytes. For example : #42048
    #4: Indicates that the length field is 4 digits long.
    2048: Specifies that the data block is 2048 bytes long (with termination characters).

    Block data
    Float value as 32 bit (4 bytes) hex value.

    Termination characters
    "\n" for HiSLIP, "\r\n" for RAW TCP and UDP.

    NOTE : The format commands influence the query commands in Measurement Commands and UDP Commands groups.

UDP Commands

Commands available only over RAW UDP.

  • UDP:INITiate[:IMMediate]
    Start measurements. You can store up to 32,000 measurements in the reading memory.
    To retrieve the measurements from the reading memory, use FETCh? or DATA[:DATA]? in RAW TCP or HiSLIP mode. Use UDP:FETCh? for RAW UDP.
  • UDP:FETCh?
    Read stored measurement data. This command will not start a new measurement.
  • UDP:MEASure?
    Measures AC or DC current.
  • UDP:READ?
    Measures AC or DC current.
  • UDP:R?
    Measures AC or DC current.

Calibration Commands

Perform device calibration.

  • CALIBration:ADC:AUTO[:MODE] {OFFSet|LINEARity}
    Perform the automatic ADC calibration. The calibration is being perform automatically during power on.

    OFFSet - offset-error cancelling.
    LINEARity - compensation of capacitance mismatch.
  • CALibration:COUNt?
    You can query the instrument to determine how many calibrations have been performed. This number is incremented when the calibration data is being stored in the EEPROM.
  • CALibration:VALue <value>
    CALibration:VALue?
    Input the reference value provided by a calibration standard.

    <value> - valid float value that corresponds to the expected measurement range of the instrument.

    NOTE : The device needs to be unsecured by using the SYSTem:SECure:STATe command.
  • CALibration:STORe
    Store the calibration values to the device EEPROM.

    NOTE : The device needs to be unsecured by using the SYSTem:SECure:STATe command.
